Проєкт було створено для участі в хакатоні 2026 року

Основною проблемою було визначити, які точки належать штату Нью-Йорк, а які - ні

Спочатку було відсіяно точки, які поза штатом, потім було створено файл (за допомогою відкритих джерел) в якому були всі zip коди, декілька ReportingCode (які встигли знайти за наданий час, тому він є не у всіх), 
штати, округи, міста, координати центрів zip-кодів та головне податки для кожного з штатів, міст, округів та спеціальний податок.

Основна логіка така:

1. Користувач завантажує CSV файл
2. Файл проходить перевірку чи всі точки належать штату Нью-Йорк, якщо ні - видаляємо
3. Далі беремо координати і шукаємо найменшу відстань до центру координати zip кода
4. Присвоюємо код (ReportingCode), штат, округ, місто і податки відповідно до них
5. Створюється таблиця де ці дані записуються і передаються безпочиредньо на html сторінку для показу

Це була логіка для CSV файл завантаження, воно розуміє під собою, що файл завантажили раз, подали податковій і він більше не є необхідний, або якзо прийшла податкова, 
то вони відкрили файл який їх цікавить та перевірили дані (адже в нашому випадку податкова буде цікавиться саме попередніми)

Якщо говоримо за ручне створення, то логіка пошуку податків не змінна, але тут ми вже створюємо тимчасовий файл в який все записуємо і виконуємо дії і основний для довготривалого зберігання
